# CLAUDE.md

이 파일은 `roi-corner-detection-ver2` 프로젝트에서 문서와 코드를 작성할 때 따르는
규칙이다.

## 1. 문서 작성 규칙

- em dash, 유니코드 화살표, 이모지를 사용하지 않는다.
- 마크다운 본문에서 화살표는 `$\to$`를 사용한다. fenced code block과 inline code 안에서는
  ASCII `->`를 사용한다.
- 폴더 구조 트리는 `├ ─ │ └` 문자를 사용한다. 화살표를 사용하지 않는다.
- 폴더/파일 목록은 폴더를 알파벳순으로 먼저 나열하고, 그다음 파일을 알파벳순으로
  나열한다.
- Jupyter 노트북 셀의 `source` 배열 마지막 원소는 줄바꿈으로 끝나지 않는다.

## 2. 코드 작성 규칙

- 코드에 한국어를 사용하지 않는다. 식별자, 주석, 문자열은 모두 영어로 작성한다.
- 세로 정렬(컬럼 정렬)을 위한 공백을 넣지 않는다.
- 경로 처리는 `pathlib.Path` 대신 `os.path`를 사용한다.
- 타입 힌트를 사용하지 않는다.
- 모든 파일의 첫 줄은 `# path/from/project/root.py: one-line description` 형식의
  헤더로 시작하고, 빈 줄 하나를 두고 import를 작성한다.
- 클래스와 최상위 함수는 1줄 docstring을 작성한다. 메서드에는 docstring을 작성하지
  않는다.
- 주석은 필요한 경우에만 최소한으로 작성한다.
- `src/` 하위 모든 폴더에는 빈 `__init__.py`를 둔다.
- `src/` 내부 모듈 간 import는 `src.xxx` 형태의 절대 경로만 사용한다. `.`, `..` 상대
  import는 사용하지 않는다.
- `scripts/`, `experiments/` 등 `src/` 외부 코드에서는 `sys.path`에 프로젝트 루트를
  추가한 뒤 `src.xxx` 형태로 import한다.
